# 调研需求定义

## 项目背景
团队正在迭代一个音乐交友产品（Apple Music 播放列表 → 音乐人格 → 匹配聊天 → 信息流广场），同时探索新的 AI 产品机会。

现有业务包括：
- **Gravity** — 日本市场社交语音房产品
- **synclub** — 虚拟角色陪伴产品
- 音乐交友产品（迭代中）

注意：Gravity 和 synclub 不在本报告分析范围内，但其运营经验（日本市场、语音社交、虚拟角色）构成团队已有认知。

## 报告结构

### Part 1: AI × 音乐社交
- **01-landscape.md** — 市面上 AI × 音乐的产品形态扫描（含独立 App + Discord bots + 开源/API 能力层）
- **02-social-plays.md** — AI × 音乐在社交/交友场景的具体玩法拆解
- **03-taste-as-product.md** — "音乐品味"如何从消费数据变为可感知的审美判断，并形成产品传播卖点

### Part 2: 新 AI 产品机会（排除音乐×社交，不排除 AI×社交）
- 市场信号概览（Revenue/MAU 跨赛道对比 + 筛选逻辑）
- **AI 增强型机会** — 现有品类 + AI 体验升级（如 AI 短视频融合、AI 短剧、AI 垂直工具）
- **AI 创新型机会** — AI 使全新产品形态成为可能（如 AI 原生社交、AI 自我表达、AI 陪伴进化）
- 每个机会深度展开隐含回答：解决谁的什么问题、为什么能增长、怎么做增长、可行落地路径

### 全文概览（含能力平台假设，不单独成章）
- ToC 业务战略方向选择
- 通用能力层定义（跨产品复用什么）
- 场景切入选择（从哪切入 + 原因）
- 组合产品间的协同与边际成本递减逻辑

## 产出标准

### 每个机会必须包含
1. 市场规模概算（TAM/SAM/SOM + 推导链）
2. 竞品分析（主要玩家、产品形态、商业模式、规模数据）
3. 速度到规模评估（收入/用户/传播路径）
4. Why Now（时机信号）
5. 可行落地路径判断
6. 防御性分析

### 筛选标准（通过第一层扫描的门槛）
- C 端应用，结合 AI 能力（不限于自称"AI 产品"的应用，关注实际玩法中嵌入 AI 能力的高增长产品）
- 优先：AI 重构具体场景的任务完成方式或产出结果
- 次优：AI 让贵价/定制化过程可大范围普及
- 关键维度：能够在几个月内快速做大收入规模/用户规模/形成口碑传播
  - 快速收入：用户已有预算、价值感知即时、高频或高客单
  - 快速用户：零摩擦上手、单人闭环、借力已有平台分发
  - 快速口碑：产出物天然可分享、wow moment 可见、社交货币属性

### 报告定位
- 外部信息的客观呈现 + 外部机会/竞品公司的路径判断
- 团队能力边界不纳入报告，作为读后内部讨论事项
- 面向团队决策使用

## 数据使用原则
- **可引用（纳入报告）**：Sensor Tower API 原始返回数据（可溯源：有 app_id、query 参数、时间戳）、公开可查的第三方数据源
- **仅参考（需验证后才纳入）**：reference 中 AI 对话导出的二次分析结论和推导链
- 报告中任何数据点需注明来源

## 数据来源
- Sensor Tower API（auth_token 已配置）
- 公开市场报告与行业分析
- 产品公开信息（App Store、官网、社交媒体）

## 核心指标体系
| 指标 | 说明 |
|------|------|
| MAU | 月活跃用户 |
| Revenue/MAU | 单用户收入效率 |
| RPD | 每次下载收入 |
| Download Growth % | 下载增速（环比） |
| Day 7/30 Retention | 留存率 |
| Revenue Growth % | 收入增速 |
